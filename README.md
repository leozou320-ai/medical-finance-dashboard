# Medical Finance Dashboard｜医疗投融资数据看板

> 一个面向医疗一级市场的可视化数据终端，用于快速洞察细分领域投融资趋势、活跃机构和赛道机会。

在线访问（GitHub Pages）  
👉 https://leozou320-ai.github.io/medical-finance-dashboard/

---

## 1. 项目背景 & 目标

医疗一级市场的信息高度分散，Excel 表、PDF、新闻稿、公众号推文、圈内小道消息交织在一起，难以形成**可视化、可迭代、可协作**的数据库。

本项目希望做三件事：

1. **沉淀一份结构化的医疗投融资主库**（按公司、轮次、机构、赛道、地域等维度）  
2. **提供一个开源的可视化看板**，让任何人都能通过浏览器直接查看趋势与分布  
3. **建立一个开放协作机制**，欢迎行业从业者和数据爱好者一起修正、补充和优化数据

---

## 2. 功能概览

当前看板主要支持：

- 按年份/轮次/金额维度查看医疗投融资总体趋势  
- 按 **细分赛道**（如 IVD、创新器械、创新药、医疗服务等）拆分结构  
- 按 **投资机构** 统计出手数量、赛道偏好  
- 按 **公司维度** 查看关键融资事件与标签（治疗领域、地域等）

> 说明：展示的数据为结构化后的投融资记录。部分敏感字段可能经过脱敏或聚合处理。

## 3. 技术栈 & 项目结构

本项目为 **纯前端静态页面**，无后端服务，便于 GitHub Pages / 任何静态托管平台部署。

- HTML + 原生 JavaScript + CSS
- 使用浏览器本地渲染图表和交互
- 数据通过静态文件（如 `data/*.json` 或 `data/*.csv`）加载

4. 本地运行

无需复杂环境，推荐两种方式之一：

方式 A：直接双击打开（仅在不使用 Ajax 加载数据时适用）
双击打开 index.html → 用浏览器访问


如浏览器安全策略阻止本地文件访问（file://），建议使用方式 B。

方式 B：启一个简单的本地静态服务器

在项目根目录执行（任选一种）：

# Python 3
python -m http.server 8000

# 或者 Node.js 环境
npx serve .


然后在浏览器访问：

http://localhost:8000

5. 数据规范（面向贡献者）

欢迎你对数据进行 纠错 / 补充 / 丰富标签。为避免“脏数据”，请尽量遵守以下约定（如与实际表头不同，请以仓库中的数据说明为准）：

一行代表一笔融资事件

必填字段建议包含：

company_name：公司全称

round：轮次（Pre-A/A+/B/C…，未披露统一用 NA）

date：签约/披露日期，格式 YYYY-MM-DD

amount：金额（建议统一为人民币，币种不同可增加 currency 字段）

investors：主要投资机构，多个以分号或逗号分隔

赛道与标签

sector_main：一级赛道（如 IVD / 创新器械 / 创新药 / 医疗服务 等）

therapy_area_lvl1 / therapy_area_lvl2：治疗领域标签（如 心血管 / 结构心 / 肿瘤 / 消化 等），请尽量复用已有标签而不是创造新拼写。

地域

country / province / city：尽量拆分到省、市，便于可视化统计。

来源

source：可选，填写公开可查的来源链接或简要说明（公告/媒体/调研）。

若不确定某个字段填法，可以在 PR 说明中写清楚你的假设，便于 maintainer 复核。

6. 如何参与贡献（欢迎一起共建）

我们非常欢迎互联网用户一起把这个数据库打磨得更准确、更全面：

1）数据相关贡献

补充缺失的融资事件

修正公司名称、轮次、金额、机构名称等错误

增加/修正赛道标签、治疗领域标签

优化字段设计（在不破坏兼容性的前提下）

2）前端与产品相关贡献

优化图表配置、交互体验、移动端展示

新增筛选条件、维度切换（如按币种、按阶段、按是否国产/进口等）

改进性能（大数据量加载、懒加载等）

提交流程（标准 GitHub 协作模式）

Fork 本仓库到你的账户

在本地新建分支，例如：feat/add-2024q3-deals 或 fix/typo-company-name

修改数据文件或前端代码，并自测本地运行正常

提交 Commit，Push 到你自己的仓库

发起 Pull Request 到本仓库的 main 分支

在 PR 描述中简要说明：变更内容、数据来源（如有）、是否存在不确定点

如只是不确定是否有权限直接改数据，也可以先通过 Issues 提交问题或建议，由维护者统一处理。
